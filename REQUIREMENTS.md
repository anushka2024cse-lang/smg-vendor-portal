# SMG Vendor Portal - System Requirements

## 1. System Overview
The **SMG Vendor Portal (MMP)** is a comprehensive web application designed to streamline material management, vendor interactions, production planning, and administrative oversight. It serves as a central hub for managing the supply chain lifecycle.

## 2. Functional Requirements

### 2.1. Dashboard & Analytics
- **Real-time Metrics**: Display total active models, low stock alerts, and total inventory value.
- **Visual Analytics**: Interactive charts showing inventory distribution and trends.
- **Quick Navigation**: access to all major modules via a persistent sidebar.

### 2.2. Inventory Management
- **Stock Tracking**: View current stock levels for all parts.
- **Create Part**: Form to add new inventory items with details (Name, SKU, Location, Quantity).
- **Transaction History**: Tabs to view "Received" and "Dispatched" material history.
- **Export**: Ability to export inventory data to Excel.

### 2.3. Product Models
- **Model Catalog**: List of all product models with their specifications.
- **Model Creation**: Interface to define new product models, including uploading images.
- **Specifications**: Track detailed specs like Category, Series, and Dimensions.

### 2.4. Vendor Management
- **Vendor Database**: Registry of all approved vendors.
- **Vendor Onboarding**: Form to add new vendors with contact and compliance details.
- **Performance Tracking**: View history of transactions per vendor.

### 2.5. Material Operations
- **Material Receipt**: Dedicated interface for logging incoming shipments (GRN generation).
- **Material Dispatch**: Interface for processing outbound stock to production lines.
- **Serial Numbering**: mocked auto-generation of serial numbers for tracked items.

### 2.6. Production & Forecasting
- **Production Planning**: History of production runs and output logging.
- **AI Forecasting**: Module to predict stock needs based on historical usage (Mock implementation).

### 2.7. Administration (Admin Role)
- **User Management**:
    - List all system users.
    - Create new users (Name, Email, Role).
    - Edit existing user details.
    - Delete users.
- **Support Ticket Management**:
    - View incoming support tickets.
    - Reply to tickets and update status (New -> Resolved).
    - Delete resolved tickets.

### 2.8. Support Center (User Role)
- **Knowledge Base**: Collapsible FAQ section for common queries.
- **Direct Support**: Contact form to submit tickets (Technical, Account, Billing).

## 3. Non-Functional Requirements

### 3.1. User Interface (UI)
- **Theme**: High-fidelity Dark Mode compliant (slate/navy color palette).
- **Responsiveness**: Layout adapts to different screen sizes.
- **Interactivity**: Immediate feedback on actions (modals, dropdowns, hover states).

### 3.2. Technical Constraints
- **Frontend**: React 19, Vite, Tailwind CSS 4.
- **State Management**: React Hooks (useState, useEffect, useMemo).
- **Network**: Axios for API communication (configured for JWT Auth).
- **Configuration**: Centralized `config.js` for managing environments (Mock vs Live) via `.env`.

## 4. Backend Dependencies (Recommended)
To fully power this frontend, the backend must support:
- **Node.js Environment**: v14+
- **Database**: MongoDB (preferred) or PostgreSQL.
- **Authentication**: JWT (JSON Web Tokens).

### Required Backend Endpoints
- `/auth/*` (Login, Register, Me)
- `/dashboard/*` (Metrics, Charts)
- `/inventory/*` (CRUD Operations)
- `/admin/*` (User Management)
- `/vendors/*` (Vendor CRUD)
